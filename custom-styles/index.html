@layer Directive

Organize custom styles into Tailwind's cascade layers

Understanding Layers

Base

Element defaults (html, body, etc.)

Components

Reusable component classes

Utilities

Single-purpose utility classes (highest priority)

/* style.css */
@tailwind base;

/* Add custom base styles */
@layer base {'{'}
  h1 {'{'}
    @apply text-4xl font-bold mb-4;
  {'}'}
  
  h2 {'{'}
    @apply text-2xl font-semibold mb-3;
  {'}'}
{'}'}

@tailwind components;

/* Add custom components */
@layer components {'{'}
  .btn {'{'}
    @apply px-4 py-2 rounded font-semibold transition-colors;
  {'}'}
{'}'}

@tailwind utilities;

/* Add custom utilities */
@layer utilities {'{'}
  .text-shadow {'{'}
    text-shadow: 2px 2px 4px rgba(0, 0, 0, 0.1);
  {'}'}
{'}'};

Best Practices

  • ‚Ä¢ Keep custom CSS minimal

‚ùå Don't

  • ‚Ä¢ Use !important excessively
  • ‚Ä¢ Duplicate Tailwind functionality

Decision Tree: Utilities vs Custom CSS

‚Üí YES: Use utilities (90% of the time)

‚Üí NO: Consider custom CSS with @apply or @layer

Is it a reusable component?

‚Üí YES: Use @apply to create a component class

‚Üí NO: Use arbitrary values or inline styles

Is it complex CSS?

‚Üí YES: Write custom CSS with regular syntax

‚Üí NO: Use @apply to keep it organized
