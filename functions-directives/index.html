Core Tailwind Directives

@tailwind

Injects Tailwind's layers of CSS.

@tailwind base; @tailwind components; @tailwind utilities;

Place these three lines in your main CSS file. They're replaced with the actual CSS during build.

@layer

Adds CSS to a specific cascade layer.

@layer base { ... } @layer components { ... } @layer utilities { ... }

Controls CSS specificity. Components > Base for precedence, Utilities > Components for overriding.

@apply

Applies utility classes to a selector.

.btn {
@apply px-4 py-2 rounded;
} /* Result: applies px-4, py-2, rounded styles */

Mix utilities into component CSS. Useful for reducing HTML class bloat.

@supports

CSS feature detection.

@supports (display: grid) {
/* Modern styles */
}

Apply styles only if browser supports a feature. Great for progressive enhancement.

The theme() Function

Usage: theme('path.to.value')

Access theme configuration values in CSS. Replaces paths with actual values.

Colors

.header {
  background-color: theme('colors.blue.600');
  color: theme('colors.white');
}
Styled with theme('colors.blue.600')

Spacing

.card {
  padding: theme('spacing.6');
  margin-bottom: theme('spacing.4');
}

Padding: theme('spacing.6')

Margin-bottom: theme('spacing.4')

Font Sizes

.heading {
  font-size: theme('fontSize.2xl');
  line-height: theme('lineHeight.tight');
}
Heading with theme('fontSize.2xl')

Border Radius

.btn {
  border-radius: theme('borderRadius.lg');
}

üéØ Why Use theme()? It keeps your custom CSS synchronized with your Tailwind config. Change a color in the config and it updates everywhere.

CSS Variables Integration

Combining theme() with CSS Variables

Use theme() to set CSS variables in your config, then access them dynamically in HTML.

Color Variables

:root {
  --color-brand: theme('colors.indigo.600');
  --color-text: theme('colors.gray.900');
}

Styled with CSS variable

--color-brand: theme('colors.indigo.600')

Spacing Variables

:root {
  --spacing-base: theme('spacing.4');
  --spacing-lg: theme('spacing.8');
}
Base spacing: var(--spacing-base)
Large spacing: var(--spacing-lg)

Advanced CSS @-Rules

1

@media (Media Queries)

@media (max-width: theme('screens.md')) { .sidebar { display: none; } }

Apply styles based on screen size. Tailwind handles this with responsive prefixes, but you can use @media directly for complex conditions.

üëâ Resize your browser to see responsive behavior

2

@keyframes (Animations)

@keyframes slide-in { from { transform: translateX(-100%); } to { transform: translateX(0); } }

Define custom animations. Use with Tailwind's animation utilities or custom classes.

Animated element
3

@supports (Feature Detection)

@supports (display: grid) { .grid-layout { display: grid; grid-template-columns: repeat(3, 1fr); } }

Progressive enhancement: use modern CSS if supported, fallback otherwise.

4

@import (Font/Stylesheet Imports)

@import url('https://fonts.googleapis.com/css2?family=Inter'); @layer base { body { font-family: 'Inter', sans-serif; } }

Load external resources. Place in CSS file, not in HTML head.

5

@font-face (Custom Fonts)

@font-face { font-family: 'CustomFont'; src: url('/fonts/custom.woff2') format('woff2'); }

Define custom fonts from files. Use in theme or directly in CSS.

Best Practices & Tips

‚úÖ Do's

  • ‚úì Use theme() in custom CSS - Keeps styles in sync with config
  • ‚úì Use @layer for organization - Prevents specificity issues
  • ‚úì Use @supports for progressive enhancement - Works in all browsers
  • ‚úì Combine with CSS variables - For dynamic theming
  • ‚úì Prefer utilities over custom CSS - Utility classes are more maintainable
  • ‚úì Use @apply sparingly - For reusable component patterns

‚ùå Don'ts

  • ‚úó Don't use @apply with important! - Breaks Tailwind's cascade
  • ‚úó Don't nest theme() calls deeply - Use single, clear paths
  • ‚úó Don't forget to close @media blocks - Syntax errors break CSS
  • ‚úó Don't abuse @supports - Use for feature detection, not browser sniffing
  • ‚úó Don't hardcode values - Always use theme() when possible
  • ‚úó Don't override with inline styles - Use CSS for consistency
